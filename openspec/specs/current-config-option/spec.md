## 新增需求

### 需求:当前配置选项
系统必须在 Profile 下拉框最顶部提供"当前配置"选项。该选项的 value 必须为 `__current__`，显示文本为"当前配置"。

#### 场景:下拉框始终包含当前配置选项
- **当** 下拉框被刷新（初始化、保存 Profile、删除 Profile 后）
- **那么** "当前配置"必须始终作为第一个选项出现在列表顶部

### 需求:当前配置从文件实时读取
选择"当前配置"时，系统必须从 settings.json 实时读取连接配置并填充表单，而非从内存缓存读取。

#### 场景:选择当前配置
- **当** 用户从下拉框选择"当前配置"
- **那么** 系统必须调用 `api.load_settings()` 从 settings.json 读取最新值，并将连接配置字段填充到表单中

#### 场景:当前配置反映外部修改
- **当** 外部程序修改了 settings.json 且用户选择"当前配置"
- **那么** 表单必须显示 settings.json 中的最新值

### 需求:默认选中当前配置
应用启动时，下拉框必须默认选中"当前配置"。

#### 场景:应用启动
- **当** 应用启动并完成设置加载
- **那么** 下拉框必须选中"当前配置"选项

### 需求:确认生效后回到当前配置
用户点击"确认生效"后，下拉框必须自动选中"当前配置"。

#### 场景:确认生效后
- **当** 用户修改配置后点击"确认生效"且保存成功
- **那么** 下拉框必须自动选中"当前配置"选项
