## 上下文

当前 Profile 下拉框使用 HTML `<select>` 的 `change` 事件。重复点击已选中项不触发事件，导致无法重新加载同一 Profile。用户需要一个"回到文件实际值"的操作。

## 目标 / 非目标

**目标：**
- 在下拉框顶部增加"当前配置"选项，从 settings.json 实时读取
- 启动时默认选中"当前配置"
- 点"确认生效"后自动选中"当前配置"
- 解决重复点击同一 Profile 不生效的 bug

**非目标：**
- 不修改后端逻辑（前端变更即可）
- 不做"未保存修改"提示或脏标记

## 决策

### 决策 1："当前配置"的值来源

**选择**：从 settings.json 实时读取（方案 A）

**替代方案**：从内存中的 currentSettings 读取

**理由**：实时读取保证"当前配置"永远是文件的真实值。即使外部修改了 settings.json，点"当前配置"也能看到最新内容。读取操作几乎无延迟。

### 决策 2："当前配置"的实现方式

**选择**：在下拉框顶部添加一个特殊的 option，value 设为特殊标识 `__current__`

**理由**：选择 `__current__` 作为 value 不会与任何 profile filename 冲突。handleProfileSelect 检测到此值时调用 `api.load_settings()` 获取最新值。

### 决策 3：初始选中状态

**选择**：应用启动和"确认生效"后，下拉框自动选中"当前配置"

**理由**：与用户心智模型一致——"当前配置"就是当前生效的配置，应该是默认状态。

## 风险 / 权衡

- **[权衡] 每次点"当前配置"多一次 API 调用** → 读取本地文件几乎无感，可接受
- **[风险] profile filename 恰好叫 `__current__`** → 极低概率，且 profile 文件名用 kebab-case 不会含双下划线
