## 新增需求

### 需求:启动桌面窗口
系统必须通过 pywebview 创建原生桌面窗口，启动方式为双击 .pyw 文件或执行打包后的 .exe。窗口必须无浏览器地址栏，内嵌 HTML 页面作为 UI。

#### 场景:双击启动
- **当** 用户双击 main.pyw 文件
- **那么** 系统弹出原生桌面窗口，加载配置编辑界面

#### 场景:窗口尺寸
- **当** 窗口启动
- **那么** 窗口尺寸必须为 800x600，窗口标题为 "Claude Settings Manager"

### 需求:读取配置
系统必须读取 `~/.claude/settings.json` 并将内容渲染到 UI 表单中。读取时必须保留所有未知字段，不被解析逻辑丢弃。

#### 场景:首次打开
- **当** 工具启动且 settings.json 存在
- **那么** 系统将 env、enabledPlugins、extraKnownMarketplaces 的所有字段填充到对应表单控件中

#### 场景:配置文件不存在
- **当** 工具启动且 settings.json 不存在
- **那么** 系统必须显示空表单（使用默认值），用户可以创建新配置

#### 场景:配置文件包含未知字段
- **当** settings.json 包含工具未识别的字段
- **那么** 系统必须在保存时原样保留这些字段，不被覆盖或丢失

### 需求:编辑连接配置
系统必须提供表单控件用于编辑 `env` 中的以下连接配置字段：ANTHROPIC_BASE_URL、ANTHROPIC_AUTH_TOKEN、ANTHROPIC_DEFAULT_OPUS_MODEL、ANTHROPIC_DEFAULT_SONNET_MODEL、ANTHROPIC_DEFAULT_HAIKU_MODEL、API_TIMEOUT_MS。

#### 场景:编辑 API 端点
- **当** 用户修改 ANTHROPIC_BASE_URL 输入框的值
- **那么** 系统记录修改，尚未写入文件

#### 场景:编辑认证令牌
- **当** 用户修改 ANTHROPIC_AUTH_TOKEN 输入框的值
- **那么** 令牌字段必须默认以掩码显示，用户可点击切换明文/掩码

#### 场景:编辑模型映射
- **当** 用户修改任一 ANTHROPIC_DEFAULT_*_MODEL 输入框的值
- **那么** 系统记录修改，尚未写入文件

#### 场景:编辑超时时间
- **当** 用户修改 API_TIMEOUT_MS 输入框的值
- **那么** 系统仅允许输入数字

### 需求:编辑偏好开关
系统必须提供复选框控件用于编辑 `env` 中的以下开关字段：DISABLE_TELEMETRY、DISABLE_ERROR_REPORTING、CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC。值为 "1" 表示启用（勾选），字段不存在或值为其他表示禁用。

#### 场景:切换遥测开关
- **当** 用户勾选或取消勾选 DISABLE_TELEMETRY 复选框
- **那么** 系统记录修改，尚未写入文件

#### 场景:开关值映射
- **当** 用户勾选开关
- **那么** 保存时对应 env 字段值必须为 "1"
- **当** 用户取消勾选开关
- **那么** 保存时必须移除该 env 字段

### 需求:编辑插件开关
系统必须为 `enabledPlugins` 中的每个插件提供复选框控件。值为 true 表示启用。

#### 场景:切换插件
- **当** 用户勾选或取消勾选某插件的复选框
- **那么** 系统记录修改，保存时将对应值设为 true 或从对象中移除

### 需求:编辑市场源
系统必须以只读列表方式显示 `extraKnownMarketplaces` 中的条目，展示名称和来源信息。

#### 场景:查看市场源
- **当** 工具加载 settings.json 包含 extraKnownMarketplaces
- **那么** 系统必须列出每个市场源的名称和 GitHub 仓库地址

### 需求:保存配置
系统必须将 UI 中的修改写回 `~/.claude/settings.json`，保留所有未修改的字段（包括工具未识别的字段）。写入必须使用格式化的 JSON（2 空格缩进）。

#### 场景:点击保存
- **当** 用户点击保存按钮
- **那么** 系统将当前 UI 中所有字段的值写入 settings.json，保留未知字段，显示保存成功提示

#### 场景:写入失败
- **当** 写入文件时发生权限错误或磁盘满
- **那么** 系统必须显示错误提示，不修改原文件内容

### 需求:单实例运行
系统必须防止多个实例同时运行，避免文件写入冲突。

#### 场景:尝试启动第二个实例
- **当** 已有一个实例在运行时用户再次双击启动
- **那么** 系统必须将已有实例窗口激活到前台，不启动新实例
